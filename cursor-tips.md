# Cursor AI Tips & Skills

## 🤖 如何讓 Cursor AI 更好地協助你

### 使用 @ 符號引用檔案
```
你：@README.md 這個專案有哪些工具？
AI：根據 README.md，這個專案包含...
```

### 使用 @ 符號引用資料夾
```
你：@src/ 這個資料夾的結構是什麼？
AI：src/ 資料夾包含以下檔案...
```

### 使用 @ 符號引用 skills
```
你：@git-workflow.md 如何建立 Release？
AI：根據 git-workflow.md，建立 Release 的步驟是...
```

## 💬 提問技巧

### ✅ 好的提問方式

**具體描述**
```
❌ 不好：如何處理影片？
✅ 好：如何批次處理 C:\videos 資料夾的影片並跳過已有字幕的檔案？
```

**提供上下文**
```
❌ 不好：出錯了
✅ 好：執行 split_whisper_merge_v13.ps1 時出現 "CUDA out of memory" 錯誤，
      我有 RTX 3080 (10GB)，目前設定是 -WorkersMap "0=2"
```

**分步驟詢問**
```
❌ 不好：幫我做所有事
✅ 好：
    1. 先幫我生成 PDF 的 glossary
    2. 然後用 glossary 處理影片
    3. 最後檢查字幕品質
```

## 📝 Composer 模式技巧

### 使用 Composer（Ctrl+I）執行複雜任務

**範例 1：多步驟操作**
```
你：請執行以下步驟：
    1. 讀取 C:\videos 資料夾的所有 MP4 檔案
    2. 對每個檔案生成 SRT 字幕
    3. 檢查 SRT 品質並輸出報告
```

**範例 2：修改多個檔案**
```
你：@src/ 將所有 PowerShell 腳本的錯誤處理改為：
    $ErrorActionPreference = "Stop"
    並加上 try-catch
```

## 🔧 Cursor 快捷鍵

### 常用快捷鍵
```
Ctrl+K        - 開啟 AI Chat
Ctrl+I        - 開啟 Composer
Ctrl+L        - 清除對話
Ctrl+Shift+P  - 指令面板
Ctrl+P        - 快速開啟檔案
Ctrl+`        - 開啟終端機
```

### AI 輔助編輯
```
Ctrl+K        - 在編輯器中開啟 AI 編輯
Tab           - 接受 AI 建議
Esc           - 拒絕 AI 建議
```

## 📚 Skills 檔案最佳實踐

### 保持 Skills 檔案更新
當你發現新的解決方法或技巧時：
```
你：我剛發現 GPU 記憶體不足時，可以用 -SegParts 10 增加分段數量，
    請幫我加入 troubleshooting.md
```

### 專案特定的 Skills
在每個專案根目錄建立 `skill.md`，記錄專案特定的：
- 常用指令
- 特殊工作流程
- 已知問題和解決方法

### 通用 Skills（這個資料夾）
在 `my-skills` 資料夾記錄通用的：
- Git 工作流程
- PowerShell 技巧
- 疑難排解方法

## 🎓 AI 協助的層級

### Level 1：查詢資訊
```
你：這個專案有哪些工具？
你：如何執行某個指令？
```

### Level 2：解決問題
```
你：遇到某個錯誤，如何解決？
你：為什麼 GPU 沒有被使用？
```

### Level 3：修改程式碼
```
你：幫我加入一個新功能
你：重構這段程式碼
```

### Level 4：複雜任務
```
你：建立一個完整的自動化流程
你：設計一個新的工具
```

## 🚫 注意事項

### 不要做的事

**❌ 模糊的請求**
```
不好：幫我處理
不好：修一下
不好：優化程式碼
```

**❌ 沒有提供足夠資訊**
```
不好：出錯了（沒有錯誤訊息）
不好：不能用（沒有說明什麼不能用）
```

**❌ 要求修改但沒說明原因**
```
不好：把這個改成那個（為什麼？）
```

### 應該做的事

**✅ 明確的請求**
```
好：幫我處理 C:\videos 資料夾的所有 MP4 檔案
好：修復 line 42 的語法錯誤
好：優化 GPU 使用率，目前只有 50%
```

**✅ 提供完整資訊**
```
好：執行指令時出現以下錯誤訊息：[完整錯誤]
好：我想實作 [功能]，目前遇到 [問題]
```

**✅ 說明需求和原因**
```
好：我想把 commit message 改為中文，
    因為團隊都使用中文溝通
```

## 📋 檢查清單

在請求 AI 協助前：

- [ ] 我的問題描述清楚嗎？
- [ ] 我有提供必要的上下文嗎？
- [ ] 我有指定相關的檔案或資料夾嗎？（使用 @）
- [ ] 我有說明預期的結果嗎？
- [ ] 如果有錯誤，我有提供完整的錯誤訊息嗎？

## 🎯 實際範例

### 範例 1：處理新專案
```
你：我有一個新專案在 C:\new-project，
    請幫我：
    1. 初始化 git
    2. 建立 .gitignore
    3. 加入 @my-skills/git-workflow.md 作為 submodule
    4. 建立基本的 README.md
```

### 範例 2：解決複雜問題
```
你：@troubleshooting.md GPU 記憶體不足的解決方法試過了，
    還是出錯。目前設定：
    - GPU: RTX 3080 10GB
    - WorkersMap: "0=1"
    - Model: medium
    - SegParts: 10
    錯誤訊息：[貼上完整錯誤]
    還有其他方法嗎？
```

### 範例 3：學習和記錄
```
你：我發現一個很好用的技巧：
    在處理長影片時，用 -SegSec 120 比預設的 180 更穩定，
    記憶體使用也更平均。
    請幫我加入 @cursor-tips.md 的「最佳實踐」章節
```
